3. 프로젝트 상세
문제 정의

기존 AI 비서는 지나치게 수동적이어서 일정 관리나 메모 등 실제 비서 역할을 충분히 수행하지 못한다고 느꼈습니다. 이에 사용자의 맥락을 파악하고 능동적으로 행동하는 AI 비서를 만들고자 했습니다.

아키텍처/설계

음성 인식 모듈과 OpenAI 기반 LLM을 결합해, 발화 → 텍스트 변환 → 의도 파악 → 툴 호출 → 응답(텍스트/음성)까지의 파이프라인을 구성했습니다.

능동적 행동을 위해 프롬프트 설계를 중점적으로 다듬고, 제공할 기능들의 구조(일정, 메모, 정보 탐색 등)를 모듈화했습니다.

4. 핵심 기능
기능 1. 음성 인식 기반 상호작용

요약: OpenAI Whisper를 사용해 음성 → 텍스트 변환을 수행합니다. 다국어 인식 및 번역을 지원합니다.

설명: Python으로 음성 입력을 전처리해 프롬프트로 변환하고, AI API에 매개변수로 전달하여 후속 동작(일정 등록, 메모 작성, 정보 요약 등)을 트리거합니다.

기능 2. 의도 인식 및 엔티티 추출(LLM)

요약: 사용자의 의도를 파악하고 필요한 엔티티(시간, 장소, 대상 등)를 추출합니다.

설명: 발화의 문맥에서 의도를 판별해 해당 툴(예: 캘린더, 메모, 검색)을 호출하고, 실행 결과를 텍스트 + 음성 형태로 사용자에게 전달합니다.

5. 나의 역할 및 기여

전체 플로우(음성 인식 → 의도 파악 → 툴 실행 → 응답)의 설계와 통합을 담당했습니다.

프롬프트를 반복 개선하며 오작동 케이스를 줄이고, 실제 사용 시나리오에 맞춘 수정/튜닝을 진행했습니다.
