프로젝트: 내손안의 작은 AI ‘내아이’

문제 정의

기존 AI 비서는 지나치게 수동적이어서 일정 관리나 메모 등 실제 비서 역할을 충분히 수행하지 못한다고 느꼈습니다. 이에 사용자의 맥락을 파악하고 능동적으로 행동하는 AI 비서를 만들고자 했습니다.

아키텍처/설계

  • 음성 인식 모듈과 OpenAI 기반 LLM을 결합한 파이프라인
  • 발화 → 텍스트 변환 → 의도 파악 → 툴 호출 → 응답(텍스트/음성)
  • 능동적 행동을 위한 프롬프트 설계와 기능 모듈화(일정, 메모, 정보 탐색)

핵심 기능

  • 기능 1. 음성 인식 기반 상호작용: Whisper로 다국어 음성→텍스트, 번역 지원
  • Python 전처리 → 프롬프트 변환 → AI API 매개변수 → 일정/메모/요약 실행
  • 기능 2. 의도 인식 및 엔티티 추출: 의도 파악, 시간·장소·대상 등 엔티티 추출
  • 컨텍스트 분석 → 적합한 툴 호출 → 결과를 텍스트+음성으로 전달

나의 역할 및 기여

  • 음성 인식 → 의도 파악 → 툴 실행 → 응답의 전체 플로우 설계/통합
  • 프롬프트 반복 개선으로 오작동 케이스 감소 및 시나리오 별 튜닝

결과 및 성과

  • 일정/할 일을 잊지 않도록 도와 마감 준수에 기여
  • 정보 탐색 시간을 단축해 전반적인 작업 효율 향상

회고 – 배운 점

  • LLM이 제대로 작동하도록 하는 프롬프트 작성 방식
  • 효율적 구동을 위한 파이프라인 개선 방법

회고 – 개선점

  • 기능 증가에 따른 지연 시간 문제
  • 경량화 및 모델/파이프라인 최적화로 응답 속도 개선

예시 프롬프트

사용자: 내일 오전 10시에 회의 일정 등록해 줘 그리고 30분 전에 리마인드해줘 시스템(내아이): 사용자의 발화를 텍스트로 전환 → 의도 파악(일정 등록, 리마인드 설정) → 캘린더/알림 툴 호출 → 확인 응답(텍스트+음성)
